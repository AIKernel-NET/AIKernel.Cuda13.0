# Package Distribution

AIKernel.Cuda13.0.Libtorch2.12.win-x64 uses split distribution because the full
CUDA Capability snapshot is larger than NuGet.org package size limits.

## Artifacts

| Artifact | Location | Contents | Purpose |
| --- | --- | --- | --- |
| Metadata package | NuGet.org | Managed AIKernel dependencies, README, LICENSE | Discovery and dependency metadata |
| Full runtime package | GitHub Release asset | Managed assembly, native bridge, LibTorch/CUDA DLLs, third-party notices | Actual CUDA execution |

Both artifacts use the same package id and version:

```text
AIKernel.Cuda13.0.Libtorch2.12.win-x64
```

## Metadata Package

The metadata package is the only package pushed to NuGet.org. It intentionally
does not contain:

- `lib/`
- `runtimes/`
- `native/`
- LibTorch binaries
- CUDA or cuDNN binaries
- `libtorch_bridge.dll`

It does contain dependency metadata for:

- `AIKernel.Abstractions`
- `AIKernel.Core`
- `AIKernel.Kernel`
- `AIKernel.Dtos`
- `AIKernel.Enums`

The metadata package is generated from:

```text
packaging/meta/AIKernel.Cuda13.0.Libtorch2.12.win-x64.nuspec.template
```

The release workflow replaces `$version$` with the release version and packs the
resulting nuspec.

## Full Runtime Package

The full runtime package is generated by the package project:

```text
src/AIKernel.Cuda13.0.Libtorch2.12.win-x64/AIKernel.Cuda13.0.Libtorch2.12.win-x64.csproj
```

It is uploaded to the matching GitHub Release as a release asset. It includes:

- `lib/net10.0/AIKernel.Cuda13.0.Libtorch2.12.win-x64.dll`
- `runtimes/win-x64/native/libtorch_bridge.dll`
- LibTorch 2.12.0 CUDA 13.0 runtime DLLs
- CUDA redistributable DLLs included in the LibTorch package
- `LICENSE`
- `README.md`
- `RELEASE_NOTES.md`
- `LICENSE-THIRD-PARTY/*`

Do not push the full runtime package to NuGet.org.

## Install For CUDA Execution

Download the full `.nupkg` from the matching GitHub Release, place it in a local
folder, then add that folder as a local NuGet source.

```powershell
$feed = "$HOME\.aikernel\cuda-packages"
New-Item -ItemType Directory -Force $feed | Out-Null

# Copy or download AIKernel.Cuda13.0.Libtorch2.12.win-x64.0.0.5.nupkg into $feed.
dotnet nuget add source $feed --name AIKernel-CUDA
dotnet add package AIKernel.Cuda13.0.Libtorch2.12.win-x64 --version 0.0.5 --source $feed
```

Use `--source` so NuGet selects the full runtime package from the local source
instead of the metadata package from NuGet.org.

GitHub Release URLs are not NuGet v3 package sources. Do not use a release page
URL directly with `dotnet nuget add source`.

## Release Checklist

1. Publish AIKernel managed dependencies first.
2. Ensure the self-hosted Windows runner has CUDA Toolkit 13.0.x and LibTorch
   2.12.0 CUDA 13.0.
3. Run the release workflow.
4. Verify the full runtime `.nupkg` is attached to the GitHub Release.
5. Verify the metadata `.nupkg` is published to NuGet.org.
6. In a clean consumer project, install from the local full-package source with
   `--source`.

## Architecture Boundary

This repository is an external Capability snapshot for Windows `win-x64`, CUDA
13.0, and LibTorch 2.12.0. AIKernel.Core remains CUDA-free. Other CUDA,
LibTorch, OS, or RID combinations should be separate Capability repositories.
